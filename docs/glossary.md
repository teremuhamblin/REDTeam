# 📘 Glossaire — REDTeam  
**Termes, concepts et acronymes utilisés dans le projet**

Ce glossaire centralise les définitions essentielles liées aux opérations Red Team, à la cybersécurité offensive et aux outils du framework **REDTeam**.

---

## 🔥 A — C
- **Active Recon** : Collecte d’informations impliquant une interaction directe avec la cible.
- **Attack Surface** : Ensemble des points exploitables d’un système.
- **Beacon** : Agent discret utilisé pour maintenir une communication avec un C2.
- **C2 (Command & Control)** : Infrastructure permettant de piloter des implants.
- **Credential Dumping** : Extraction de secrets (hashs, tokens, mots de passe).

---

## 🛰️ D — F
- **Defense Evasion** : Techniques visant à contourner les mécanismes de sécurité.
- **Enumeration** : Phase de collecte d’informations détaillées sur un système.
- **Exfiltration** : Extraction de données hors du périmètre cible.
- **Footprinting** : Cartographie initiale de la cible.

---

## 🧩 G — L
- **Lateral Movement** : Déplacement d’un système compromis vers un autre.
- **LOLBins** : Outils natifs du système détournés à des fins offensives.
- **Log Poisoning** : Manipulation des logs pour masquer des traces.

---

## 🛠️ M — P
- **MITRE ATT&CK** : Framework de classification des techniques offensives.
- **OPSEC** : Ensemble des pratiques visant à réduire les risques d’exposition.
- **Persistence** : Techniques permettant de maintenir un accès durable.
- **Phishing Simulation** : Campagne contrôlée visant à tester la résilience humaine.

---

## 🧬 R — Z
- **Reconnaissance Passive** : Collecte d’informations sans interaction directe.
- **Rules of Engagement (RoE)** : Cadre légal et opérationnel d’une mission.
- **TTPs** : Tactiques, Techniques et Procédures utilisées par un acteur.
- **Zero Trust** : Modèle de sécurité basé sur la vérification systématique.
